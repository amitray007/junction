# Method File 01.5 — Duplication & Boundary Tooling (Increment 1.5)

> **Lands after increment 1 (the skeleton exists), before increment 2.** Wires the tooling that enforces low duplication + module boundaries, per `docs/principles/`. Mirrors how 0.5 wired enforcement.
>
> **Builder:** Sonnet (config + small scripts). Use `/ce-work` if helpful. Mostly config — but verify each tool actually runs against the real (skeleton) workspace.

---

## Part 1 — Spec (what & why)

### Goal

Add three duplication/boundary tools to the **CI / pre-push tier** (not the inner per-edit/pre-commit loop — keep that fast). These turn `docs/principles/` into machine checks: dependency direction + **cycle detection**, version consistency, and copy-paste detection.

### Deliverables

| Tool | Role | Tier | Config |
|---|---|---|---|
| **dependency-cruiser** `^17` | Dep direction + **circular-dep detection** + orphans + over-extraction smells | pre-push / CI | `.dependency-cruiser.cjs` |
| **syncpack** `^15` (+ pnpm catalogs) | One version per dep across packages; catalog references | pre-commit + CI | `.syncpackrc` + `pnpm-workspace.yaml` catalog |
| **jscpd** `^5` | Copy-paste detection — a signal, not auto-fail | CI only, thresholded | `.jscpd.json` |

### Proof of done

- `pnpm depcruise` runs and passes on the skeleton; a deliberately-added cycle or reverse-import is **flagged**.
- `pnpm syncpack:check` passes; a deliberately-mismatched dep version is flagged.
- `pnpm dup` (jscpd) runs, reports cleanly on the skeleton, ignores `dist/`/migrations/`*.test.ts`.
- These are added to the **pre-push lefthook stage** (and CI when CI exists), **not** pre-commit (except fast syncpack check). The per-edit/pre-commit inner loop stays fast.
- `pnpm verify` still passes and stays fast (these are NOT added to `verify`).
- Committed.

### Out of scope

- sheriff (overlaps dependency-cruiser — we picked one).
- Adding these to the per-edit hook or the core `verify` gate (would slow the inner loop).

## Part 2 — Implementation

### Step 1 — dependency-cruiser

- `pnpm add -Dw dependency-cruiser@^17`.
- Run `pnpm exec depcruise --init` to generate a config in the **current recommended format** for v17 (it may emit `.dependency-cruiser.cjs` or `.js`/`.mjs` depending on the repo's module setup — accept whatever `--init` produces; don't force `.cjs` if v17 prefers another form). Then add junction's rules to it, encoding `docs/principles/modularity.md`:
  - **forbidden:** `no-circular` (error) — at module level inside `core` and across packages.
  - **forbidden:** reverse imports into `core` (anything in `packages/cli|web|mcp` must not be imported by `packages/core`).
  - **forbidden:** `cli`/`web`/`mcp/*` importing each other (only `core` is shared).
  - **forbidden:** deep imports into another package's `src` internals (must use the package entry).
  - **warn:** orphan modules; modules in a "shared" position used by only one consumer (over-extraction smell).
- Add script: `"depcruise": "depcruise packages --config .dependency-cruiser.cjs"`.
- Test: temporarily add a `core`→`cli` import and a cycle; confirm both are flagged; revert.

### Step 2 — syncpack + pnpm catalogs

- `pnpm add -Dw syncpack@^15`.
- Introduce a **pnpm catalog** in `pnpm-workspace.yaml` for deps shared across packages (e.g. `zod`, `neverthrow` once more packages use them), so the version is defined once and packages reference `catalog:`.
- Create `.syncpackrc` enforcing: one version per dependency across the workspace; consistent ranges.
- Scripts: `"syncpack:check": "syncpack lint"`, `"syncpack:fix": "syncpack fix-mismatches"`.
- syncpack's check is fast — add it to the **pre-commit** lefthook stage *and* pre-push.
- Test: set a mismatched version in one package; confirm flagged; revert.

### Step 3 — jscpd

- `pnpm add -Dw jscpd@^5`.
- Create `.jscpd.json`: conservative `minTokens`/`minLines`; **ignore** `**/dist/**`, generated Drizzle migrations (`**/drizzle/**` or wherever migrations land), `**/*.test.ts` fixtures, `node_modules`. Output a report; set a threshold but treat as a **signal** (don't wire as a hard pre-commit fail).
- Script: `"dup": "jscpd packages"`.
- Run on the skeleton — should report ~0 clones. Document in `docs/principles/dry.md` that a jscpd hit is a "rule-of-three moment arrived" prompt, not an auto-fail (already noted there).

### Step 4 — Wire into lefthook pre-push AND the CI `quality` job

**lefthook** (note: `lefthook.yml` is now v2 format — extend the existing file, don't rewrite to v1 schema): add to the existing `pre-push` stage (which already runs `pnpm verify`): `pnpm depcruise`, `pnpm syncpack:check`. Add `syncpack:check` to the existing `pre-commit` stage (it's fast). **Do not** add `depcruise`/`dup` to pre-commit or to `verify`.

**CI:** the root `"quality"` script already exists as a placeholder (`pnpm audit --audit-level=high || true`) and is run by the `quality` CI job (`.github/workflows/ci.yml`). **Expand it** to: `"quality": "pnpm exec knip || true && pnpm depcruise && pnpm syncpack:check && pnpm dup && pnpm audit --audit-level=high || true"` — wait: knip isn't installed yet (it's a separate later add per the spec). For THIS increment, set `"quality": "pnpm depcruise && pnpm syncpack:check && pnpm dup && (pnpm audit --audit-level=high || true)"`. depcruise/syncpack should FAIL the job on violation; jscpd (`dup`) and audit are signal-only (`|| true`). This makes the CI `quality` job actually enforce the boundary rules.

### Step 5 — Verify & commit

- Run each script; confirm pass on the skeleton.
- Confirm `pnpm verify` is unchanged and still fast.
- `git add -A && git commit` (pre-commit runs verify + syncpack).
